# US API - Quick Start Guide

## Main Endpoint

```
POST http://3.137.131.89:5667/api/us/report
Content-Type: application/json
```

## Request Body

```json
{
  "mrn": "12345678",
  "acc": "ACC001234",
  "report": "Report content here..."
}
```

**Fields:**
- `mrn` *(required)*: Medical Record Number
- `acc` *(optional)*: Accession Number
- `report` *(required)*: Report text content

## Response

**Success (201 Created or 200 Updated):**
```json
{
  "success": true,
  "action": "created",
  "guid": "a1b2c3d4-...",
  "mrn": "12345678",
  "acc": "ACC001234",
  "createdon": "2026-02-24T10:30:00",
  "updatedon": "2026-02-24T10:30:00",
  "message": "US report created successfully"
}
```

**Error (400):**
```json
{
  "success": false,
  "error": "Field \"mrn\" is required and cannot be empty"
}
```

## Code Examples

### Python

```python
import requests

def send_us_report(mrn, accession, report_text):
    url = "http://3.137.131.89:5667/api/us/report"
    data = {
        "mrn": mrn,
        "acc": accession,
        "report": report_text
    }
    
    response = requests.post(url, json=data, timeout=30)
    return response.json()

# Usage
result = send_us_report("12345678", "ACC001234", "Complete report...")
print(f"GUID: {result['guid']}")
```

### curl

```bash
curl -X POST http://3.137.131.89:5667/api/us/report \
  -H "Content-Type: application/json" \
  -d '{
    "mrn": "12345678",
    "acc": "ACC001234",
    "report": "Complete ultrasound report..."
  }'
```

## Additional Endpoints

### Health Check
```bash
GET http://3.137.131.89:5667/api/health
```

### Get Reports by MRN
```bash
GET http://3.137.131.89:5667/api/us/report/{mrn}
```

### Get Specific Report
```bash
GET http://3.137.131.89:5667/api/us/report/{mrn}/{acc}
```

### Statistics
```bash
GET http://3.137.131.89:5667/api/us/stats
```

## Important Notes

- If you send a report with the same `mrn` and `acc` that already exists, it will **update** the existing report
- The `acc` field is optional but recommended to avoid duplicates
- The `guid` is automatically generated by the database
- No authentication required (internal use only)
- Recommended timeout: 30 seconds
